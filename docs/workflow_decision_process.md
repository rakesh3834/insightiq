# InsightIQ Decision Workflow

## 1. Open Mixpanel

Use event data to understand behavior before revenue:

- Views to carts.
- Carts to purchases.
- Event volume by category.
- Funnel drop-offs.
- Tracking anomalies.

Output: `artifacts/funnel_summary.csv`.

## 2. Open Tableau

Use BI extracts to scan business performance:

- Revenue by category and brand.
- Average order value.
- Cancellation rate.
- Rating and review volume.
- Segment and cohort behavior.

Output: `artifacts/tableau_dashboard_extract.csv`.

## 3. Write SQL

Validate the core KPI movement directly from the warehouse.

Output: `database/analytics_queries.sql` and `artifacts/kpi_summary.json`.

## 4. Ask Data Scientist

Run models that explain patterns:

- Isolation Forest for anomaly detection.
- Simple time-series forecast for revenue.
- KMeans for user segmentation.
- Lexicon sentiment for review polarity.
- TF-IDF/NMF topic modeling for review themes.

Outputs: `artifacts/anomalies.csv`, `artifacts/forecast.csv`, `artifacts/segment_profiles.csv`, and `artifacts/review_intelligence.csv`.

## 5. Read Reviews

Connect what users say to what they do. Review themes are aggregated by product and category, then compared with ratings, revenue, and cancellation rates.

## 6. Read Release Notes

Synthetic release notes, feature flags, experiments, and incidents provide launch context when public datasets are not available.

Generated by: `scripts/generate_synthetic_datasets.py`.

## 7. Create Presentation

The report generator creates an executive narrative with KPI evidence, model outputs, customer voice, and trade-offs.

Output: `artifacts/presentation.md`.

## 8. Decision

The recommendation engine chooses one of:

- Launch.
- Iterate.
- Roll back.
- Investigate.

Output: `artifacts/decision_memo.md`.
